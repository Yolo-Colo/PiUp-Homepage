An Over Enginnered Way To Run Some Rasberry Pis

But Why?

The purpose of this project is to keep two Rasberry Pi's running for up to 2 years nonstop. The hope is that I can keep them running for a long as possible without putting them into read only mode. I've been saying away from readonly file sytem and I see it at cheating for this project. I put a lot of thought into this project in hopes I can make the two year mark. A lot of devices out there today have a silly uptime unintentional on a single power source. Why not give it two and go until it dies.
